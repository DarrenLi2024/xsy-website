# 芯师爷官网 — 项目开发计划

## 一、项目管理框架

### 1.1 开发方法论

采用 **Scrum + Kanban** 混合模式：

| 维度 | 方案 |
|------|------|
| **迭代周期** | 双周 Sprint |
| **每日站会** | 15 分钟，同步进度和阻塞 |
| **Sprint 规划** | 每 Sprint 第一天 |
| **Sprint 回顾** | 每 Sprint 最后一天 |
| **项目管理** | GitHub Projects / Linear |
| **代码审查** | 全员 PR Review，至少 1 人 Approve |
| **分支策略** | trunk-based + feature flags |

### 1.2 分支策略

```
main          ─── 生产就绪，仅 PR 合并
  ├── staging    ─── 预发布环境，集成测试
  │     └── feature/xxx  ─── 功能分支（开发）
  │           └── fix/xxx  ─── 修复分支
  └── release/v* ─── 版本发布分支
```

### 1.3 团队角色建议

| 角色 | 人数 | 职责 |
|------|------|------|
| **前端工程师** | 2 | 前台页面 + 企业后台 + 管理后台 |
| **全栈工程师** | 1 | API、数据模型、搜索、认证 |
| **UI/UX 设计师** | 1 | 设计系统、页面设计、交互规范 |
| **项目经理** | 1 | 需求管理、进度跟踪、质量把控 |

## 二、开发阶段划分

### Phase 0：项目基础设施（3 天）

**目标**：搭建可运行的开发环境，建立工程规范

| Day | 任务 | 交付物 |
|-----|------|--------|
| 1 | 修复 `package.json` 重复依赖，统一版本管理 | 可安装的依赖 |
| 2 | 修复 `docker-compose.yml`，配置 PostgreSQL + Redis | 本地开发环境 |
| 3 | 配置工程规范链：ESLint、Prettier、Husky、commitlint | 完整代码规范 |

**验收标准**：`npm run dev` 正常启动，数据库联通，lint 生效

---

### Phase 1：数据层 + 认证系统（4 天）

**目标**：完成核心数据模型建立和用户认证体系

| Day | 任务 | 交付物 |
|-----|------|--------|
| 1-2 | Prisma Schema 核心模型（Company、Product、Article、User、Session） | Schema + 迁移文件 |
| 1-2 | 种子数据脚本 | seed.ts |
| 3-4 | JWT 认证系统（注册、登录、中间件） | API + 页面 + 中间件 |

**核心模型**：Company → Product / Article / User / Session

---

### Phase 2：前台核心页面（6 天）

**目标**：公众可见的首页、企业展示、内容阅读

| Day | 任务 | 子任务 |
|-----|------|--------|
| 1-2 | 布局系统 | 根布局、Header（导航/搜索/国际化）、Footer |
| 1-2 | **首页** | Hero 区域、统计数字、推荐企业、最新文章、活动预告、底部 |
| 3-4 | **企业模块** | 企业列表页（搜索/筛选/分页） |
| 3-4 | | 企业详情页（品牌 Banner、介绍、产品、新闻 Tab） |
| 5-6 | **内容模块** | 文章列表（分类 Tab、分页） |
| 5-6 | | 文章详情（Markdown 渲染、相关文章、分享） |
| 5-6 | | 关于我们页面 |

**关键交付**：前台可访问、SEO 标签完整、响应式适配

---

### Phase 3：增值服务模块（8 天）

**目标**：跑通评选、广告、报告三条商业变现链路

| 模块 | 天数 | 详细任务 |
|------|------|---------|
| **活动管理** | 2 | 活动列表（时间线）、活动详情（报名表单、直播嵌入）、活动回顾 |
| **硬核芯评选** | 2 | 评选首页（规则/奖项）、企业参评提交、投票系统、结果展示 |
| **广告投放** | 2 | 广告位管理、企业投放、效果追踪（曝光/点击） |
| **行业报告** | 1 | 报告列表（分类/搜索）、报告详情（预览/付费下载） |
| **招聘系统** | 1 | 职位列表（筛选）、职位详情（投递入口） |

**关键交付**：三条商业链路走通，种子数据验证

---

**目标**：入驻企业可自助管理品牌主页和各项服务

| Day | 任务 | 功能点 |

| 1 | 企业后台框架 | Layout（Sidebar 导航）、企业注册/登录 |
| 2 | 控制台 + 企业信息 | 数据总览看板、企业信息编辑 |
| 3 | 产品 + 新闻管理 | 产品 CRUD、新闻/软文发布 |
| 4 | 活动 + 招聘管理 | 活动申请、职位发布/管理 |
| 5 | 广告 + 数据 | 广告计划创建、数据看板、合作案例 |

| 3 | 内容管理 | 文章管理（含企业软文审核）、活动管理 |
| 4 | 评选 + 广告管理 | 评选配置、广告位配置、投放审核 |
| 5 | 用户 + 媒体 + 设置 | 用户管理、媒体渠道、系统设置、SEO 配置 |

---

### Phase 6：搜索 + 国际化（3 天）

| Day | 任务 | 功能点 |

---

### Phase 7：测试 + 性能优化（4 天）

| Day | 任务 | 功能点 |

| Day | 任务 | 功能点 |
|-----|------|--------|

## 五、风险管理

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| Next.js 16 + Prisma 7 兼容性 | 中 | 高 | 前期 POC 验证，降级到 Next.js 15 备选 |
| 种子数据不足导致开发阻塞 | 中 | 中 | 提前准备 Mock 数据工厂 |
| 企业用户需求理解偏差 | 高 | 中 | 每期 Sprint Review 邀请企业代表参与 |
| 图片/文件存储选型变更 | 低 | 低 | 使用 S3 兼容接口，随时切换提供商 |
| Meilisearch 部署复杂度 | 中 | 中 | 初期使用 API 排序替代，搜索模块后置 |
| 投票系统并发刷票 | 高 | 中 | Redis 限流 + 验证码 + I P 频率限制 |

## 六、团队协作指南

### 6.1 代码规范

```
提交信息格式：
  type(scope): subject

  type: feat / fix / refactor / style / test / docs / chore
  scope: comp / article / auth / award / ad / report / job / event
  subject: 中文描述

示例：
  feat(auth): 实现企业 JWT 登录
  fix(article): 修复 Markdown 代码块渲染
  refactor(comp): 重构企业列表搜索逻辑
```

### 6.2 PR 规范

```
PR 标题：[Type] 简短描述

PR 描述：
  ## 变更内容
  - xxx

  ## 测试方法
  - xxx

  ## Screenshots (可选)
```

### 6.3 Issue / Task 流转

```
Backlog → Todo → In Progress → Review → Testing → Done
                    ↓ (Blocked)
                   Blocked
```
