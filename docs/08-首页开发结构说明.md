# 芯师爷官网 — 首页开发结构说明（可直接落地）

> 目标：主站首页采用「产业媒体门户信息流 + GTC 式首屏与锚点导航」混合结构；本文件供前端按区块拆分组件、对接数据与 SEO，不包含具体视觉稿数值以外的实现约束。

---

## 1. 路由与文件位置（Next.js App Router）

| 项 | 约定 |
|----|------|
| 路由 | `/`（默认语言可根路径；若启用 i18n，则为 `/[locale]/`，本说明以 `/` 为例） |
| 页面文件 | `src/app/(public)/page.tsx` 或 `src/app/[locale]/(public)/page.tsx` |
| 布局 | 继承 `(public)/layout.tsx`：全站 Header、Footer、可选 AnnouncementBar |
| 数据获取 | 首页优先 **RSC 并行请求**（`Promise.all` 或独立 `async` 子组件），首屏关键数据避免瀑布流 |
| 渲染模式 | **SSR + 短周期 ISR**（如 `revalidate = 60`），便于头条与推荐位更新 |

---

## 2. 页面信息架构（区块顺序与锚点）

自上而下固定 **10 个逻辑区块**。粘性二级导航仅锚定 **6 个主区块**（避免过多锚点）。

| 顺序 | 区块 ID（`id` 属性） | 导航文案（中文） | 说明 |
|------|---------------------|-----------------|------|
| 0 | （无，或 `top`） | — | 全站顶栏，非锚点 |
| 1 | `section-hero` | 首页 | 全宽首屏，可不进锚点栏 |
| 2 | `section-trending` | 头条 | 置顶资讯 / 专题 |
| 3 | `section-feed` | 资讯 | 多栏目信息流 |
| 4 | `section-companies` | 企业 | 推荐企业与入口 |
| 5 | `section-events` | 活动 | 近期活动 / 峰会预告 |
| 6 | `section-awards` | 评选 | 硬核芯等入口卡片 |
| 7 | `section-topics` | 专题 | 主题聚合（工艺/车规/AI 芯片等，可配置） |
| 8 | `section-proof` | 声音 | 证言 / 数据背书（可选） |
| 9 | `section-cta` | 合作 | 页尾强 CTA（入驻 / 商务） |

**粘性二级导航（Sticky Sub Nav）** 建议展示锚点：

`头条` → `#section-trending` · `资讯` → `#section-feed` · `企业` → `#section-companies` · `活动` → `#section-events` · `评选` → `#section-awards` · `合作` → `#section-cta`

- 每个锚点区块根元素设置 `scroll-margin-top: 4.5rem`（或等于 Header + SubNav 总高度），避免滚动后被遮挡。
- 当前可视区块高亮：可用 `IntersectionObserver`（client 组件）或降级为仅点击跳转不高亮。

## 3. 全局壳层（与首页耦合的 layout 行为）

### 3.1 `AnnouncementBar`（可选）

- 条件渲染：CMS / 环境变量控制一行文案 + 链接（如「GTC Taipei 专题报道」）。
- 高度约 `40px`；存在时 SubNav 的 `top` 需叠加该高度（或 Announcement 可关闭）。

### 3.2 `SiteHeader`（全站）

- 左：Logo → `/`
- 中：主导航链接（与首页锚点可重复：资讯、企业、活动… 或链到列表页 `/articles` 等）
- 右：搜索入口、语言切换、`登录/注册` 或用户菜单
- **移动端**：汉堡菜单内包含主导航 + 锚点快捷（与 SubNav 合并避免重复可二选一）

### 3.3 `HomeStickySubNav`（仅首页或 `pathname === '/'`）

- `position: sticky; top: var(--header-height)`（若顶部有 Announcement，则 `top` = header + announcement）。
- 背景半透明 + `backdrop-blur`（与品牌规范一致即可）。
- 移动端：横向 `scroll-snap` 的 chip 列表，避免换行挤压。
- **无障碍**：导航用 `<nav aria-label="首页区块">`；当前项 `aria-current="true"`（若实现区块高亮）。

### 3.4 `SiteFooter`（全站）

- 与首页 `section-cta` 区分：Footer 放版权、备案、隐私、友情链接；**商务强 CTA 放在 `section-cta`**。

## 4. 分区块结构说明（组件拆分 + 布局 + 数据）

以下组件名建议放在 `src/components/home/` 下，页面 `page.tsx` 仅组合与传参。

---

### 4.1 `HomeHero`（`id="section-hero"`）

**定位**：GTC 式首屏 — 品牌一句话 + 副文案 + 主/次 CTA + 可选背景图或视频封面（芯师爷原创素材，勿用第三方品牌图）。

**布局**

- **桌面（≥1024px）**：两栏或居中叠字二选一  
  - 方案 A：`grid grid-cols-12`，左 `col-span-6` 文案 + CTA，右 `col-span-6` 视觉（图/视频占位）。  
  - 方案 B：单列居中，`max-w-4xl`，下接全宽视觉条。
- **平板（768–1023px）**：单列，视觉在文案下，`max-h-[40vh]` 裁剪。
- **移动（<768px）**：单列；CTA 两个按钮全宽堆叠或并排 `flex gap-2`。

**内容要素**

| 字段 | 来源 | 必填 |
|------|------|------|
| `headline` | CMS / 常量 | 是 |
| `subline` | CMS / 常量 | 是 |
| `primaryCta` | `{ label, href }` | 是，如「浏览资讯」→ `/articles` |
| `secondaryCta` | `{ label, href }` | 否，如「企业入驻」→ `/enterprise/register` |
| `backgroundMedia` | 图 URL 或视频 poster | 否 |
| `stats` | 可选 3 个数字：入驻企业数、文章数、活动数（与 `04-前端设计` 首页线框一致） | 否 |

**性能**

- 首屏图：`next/image` `priority` + 明确 `sizes`。
- 若视频：仅 `poster` + 点击再加载 video（避免自动播放拖慢 LCP）。

**无障碍**

- `h1` 仅一处，使用 `headline`。
- 背景若装饰性：`aria-hidden="true"`。

---

### 4.2 `HomeTrending`（`id="section-trending"`）

**定位**：媒体门户「头条区」— 1 大头条 + 2～3 条次要头条。

**布局**

- 桌面：`grid lg:grid-cols-12 gap-6`  
  - 大卡片 `lg:col-span-7`（封面图 + 标题 + 摘要 2 行 + meta）  
  - 侧栏 `lg:col-span-5`：`flex flex-col gap-4` 小卡片列表。
- 移动：大卡片在上，小条列表在下。

**数据**

- API：`getFeaturedArticles({ limit: 4, slot: 'home-hero' })` 或 Prisma 查询 `isFeatured + publishedAt`。
- 单条卡片字段：`slug, title, summary, coverImage, category, publishedAt, readingTime?`。

**交互**

- 整块可点：`article` 包 `Link`，注意嵌套 `a` 规范（只保留一个 `Link` 包裹）。

---

### 4.3 `HomeFeed`（`id="section-feed"`）

**定位**：多栏目资讯 — 「最新 / 深度 / 企业动态」等 Tab + 列表 + 「查看更多」。

**布局**

- 区块标题行：`h2`「资讯」+ 右侧 `Link` → `/articles`。
- Tab：`role="tablist"`，`button role="tab"`，`tabpanel` 对应列表（首屏可 SSR 默认 tab 数据，切换 tab 可用路由 query `?tab=depth` 或 client fetch）。
- 列表：桌面两列 `grid md:grid-cols-2 gap-6`；移动单列。
- 每条：左图右文或上图下文（与全局卡片规范一致）。

**数据**

- 首屏 SSR：默认 tab 各 `limit=6`；其他 tab 可懒加载（进入视口再请求或点击再请求）。

**SEO**

- 区块标题用 `h2`，列表项标题 `h3` 或保留为卡片内 `p.font-semibold`（全页 `h1` 仅在 Hero）。

---

### 4.4 `HomeCompanies`（`id="section-companies"`）

**定位**：推荐企业 + 入口到企业列表。

**布局**

- 标题行 + `Link`「全部企业」→ `/companies`。
- 桌面：`grid grid-cols-2 lg:grid-cols-4 gap-4` 企业卡片（Logo、名称、一句简介、标签）。
- 可选：首卡 `lg:col-span-2` 做「本周推荐企业」大卡。

**数据**

- `getRecommendedCompanies({ limit: 8 })`，字段：`slug, name, logo, industry, shortDescription`。

---

### 4.5 `HomeEvents`（`id="section-events"`）

**定位**：近期活动 / 峰会预告（GTC 式「Experience」简版）。

**布局**

- 横向卡片组：桌面 `grid md:grid-cols-3 gap-6`；或 `Embla` 轮播（3.5 张露边暗示可滑）。
- 每张：封面、标题、时间地点、`Link` 详情 `/events/[id]` + 次要按钮「报名」若开放。

**数据**

- `getUpcomingEvents({ limit: 6, endAfter: now })`。

---

### 4.6 `HomeAwards`（`id="section-awards"`）

**定位**：硬核芯评选等 — 规则摘要 + 入口，非完整投票页。

**布局**

- 两栏：左文案 + CTA「了解评选」「参与投票」；右视觉或历届高光数字（届数、参评企业数）。
- 移动：单列顺序：文案 → CTA → 图。

**数据**

- 静态配置或 `getActiveAwardCampaign()` 单条；无活动则整区不渲染或显示「敬请期待」占位（运营可配置）。

---

### 4.7 `HomeTopics`（`id="section-topics"`）

**定位**：主题聚合入口（类似 GTC Key Topics，但语义改为产业主题）。

**布局**

- 桌面 `grid md:grid-cols-2 xl:grid-cols-4 gap-4` 大色块卡片（每块：主题名、一句描述、`Link` 到 `/articles?topic=xxx` 或专题路由）。
- 主题列表来自 CMS 或常量配置文件 `homeTopics.ts`，便于增删。

---

### 4.8 `HomeProof`（`id="section-proof"`，可选）

**定位**：证言轮播或数据背书（入驻企业数、月访问量、合作机构 logo 墙）。

**布局**

- 证言：`carousel` + 单条 quote + 出处；`prefers-reduced-motion: reduce` 时改为静态显示第一条。
- Logo 墙：`flex flex-wrap justify-center gap-x-8 gap-y-4 opacity-80`（客户授权 logo 才可展示）。

**数据**

- 静态 JSON 或 CMS；无数据则 `return null` 不占位。

---

### 4.9 `HomeCta`（`id="section-cta"`）

**定位**：页尾强转化 — 企业入驻 + 商务合作。

**布局**

- 全宽深色条（与品牌色一致），内层 `max-w-5xl mx-auto px-4 py-16 text-center`（数值可跟设计 token）。
- 主按钮「企业入驻」、次按钮「商务合作」（`mailto:` 或 `/about#contact`）。

---

## 5. `page.tsx` 组合顺序（伪代码级）

```tsx
// src/app/(public)/page.tsx 结构顺序（示意）
export default async function HomePage() {
  return (
    <>
      <HomeHero />
      <HomeStickySubNav />  {/* 或放在 layout 中根据 route 渲染 */}
      <HomeTrending />
      <HomeFeed />
      <HomeCompanies />
      <HomeEvents />
      <HomeAwards />
      <HomeTopics />
      <HomeProof />         {/* 可选 */}
      <HomeCta />
    </>
  );
}
```

说明：`HomeStickySubNav` 也可放在 `layout` 中通过 `headers()` 或 `usePathname` 仅在 `/` 渲染；实现二选一即可，避免重复。

## 6. 响应式断点（与现有设计文档对齐）

| Token | 宽度 | 首页行为摘要 |
|-------|------|--------------|
| `sm` | 640px | Tab 可并排；卡片单列为主 |
| `md` | 768px | Feed / Companies 多列起算 |
| `lg` | 1024px | Hero 双栏；Trending 大+侧栏 |
| `xl` | 1280px | Topics 四列；更大 `max-w` 容器 |
| `2xl` | 1536px | 内容区 `max-w-[1440px] mx-auto`（与 PRD 桌面优先一致） |

容器：`max-w-[1440px] mx-auto px-4 md:px-6 lg:px-8`（数值可按设计系统微调）。

## 7. SEO 与元信息（首页专属）

| 项 | 实现要点 |
|----|----------|
| `<title>` / `description` | `generateMetadata()` 从 CMS 或环境变量读取 |
| `openGraph` / `twitter` | 使用 Hero 图或默认 OG 图 |
| JSON-LD | `WebSite` + `SearchAction`（站内搜索若已上线）；可选 `Organization` |
| `h1` | 仅 `HomeHero` 一处 |
| 结构化跳转 | 各区块「查看更多」统一带 `rel` 无需；外链加 `rel="noopener noreferrer"` |

---

## 8. 性能与观测（首页 KPI）

| 项 | 要求 |
|----|------|
| LCP 元素 | 优先 Hero 图或 Trending 首图；控制体积 WebP/AVIF |
|  below-the-fold | `next/image` 默认 lazy；轮播内非首屏图 `loading="lazy"` |
| 第三方脚本 | 首页尽量少；分析脚本 `afterInteractive` |
| CLS | 图片固定 `width/height` 或 `aspect-ratio` |

## 9. 与 PRD / 路由文档的对应关系

| PRD 首页项 | 本结构区块 |
|------------|------------|
| Hero Banner、数据统计 | `HomeHero`（stats） |
| 推荐企业 | `HomeCompanies` |
| 最新文章 | `HomeTrending` + `HomeFeed` |
| 活动预告 | `HomeEvents` |
| 广告位 | **未单独成区**：建议在 `HomeHero` 下或 `HomeTrending` 上插入 `HomeAdSlot`（`position=home_top_banner`），或 `HomeFeed` 列表每 N 条插入一行广告；实现时单独组件 + 运营配置 |
| 精选报告 | 可在 `HomeFeed` 旁加 `aside` 组件 `HomeReportsTeaser`，或第二屏 `grid` 右栏；若 MVP 可省略，在 `HomeTopics` 中链一条「报告」入口 |

## 10. 开发验收清单（首页）

- [ ] 10 个区块 DOM `id` 与 SubNav 锚点一致，`scroll-margin-top` 正确。
- [ ] 仅一个 `h1`；区块标题层级连续、无跳级。
- [ ] 键盘可聚焦 SubNav chip；Skip link「跳到主要内容」至 `#section-trending` 或 `#section-hero`。
- [ ] 移动端 SubNav 横向滚动不引起整页横向溢出。
- [ ] 无数据模块安全降级（不报错、不占畸形空白）。
- [ ] Lighthouse：Performance / Accessibility / SEO 分数达到项目门禁（见 PRD）。
