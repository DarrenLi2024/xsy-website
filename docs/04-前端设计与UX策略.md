# 芯师爷官网 — 前端设计与 UX 策略

## 一、设计语言系统

### 1.1 设计系统架构

芯师爷官网采用 **Atomic Design（原子设计）** 方法论构建设计系统：

```
原子 (Atoms)      分子 (Molecules)    组织 (Organisms)     模板 (Templates)      页面 (Pages)
Token/Color       SearchBar          Header/Navbar       Public Layout        Home Page
Typography        Card               Footer              Admin Layout         Company List
Icon              Form Field         DataTable           Enterprise Layout    Article Detail
Button            Modal              Sidebar
Input             Tabs               Hero Section
```

## 三、交互设计规范

### 3.1 微交互动效

| 场景 | 动效 | 时长 | 缓动函数 |
|------|------|------|---------|
| 按钮悬浮 | 背景色加深 + 轻微上移 | 200ms | ease-out |
| 卡片悬浮 | 阴影增强 + 向上位移 4px | 300ms | ease-out |
| 页面切换 | 淡入淡出 | 300ms | ease-in-out |
| 导航下拉 | 展开 + 透明度变化 | 200ms | ease-out |
| Toast 提示 | 从顶部滑入 | 400ms | ease-out |
| Modal 打开 | 缩放 + 淡入 | 250ms | ease-out |
| 计数动画 | 数字滚动递增 | 600ms | ease-out |

### 3.2 加载策略

```
层级加载策略：
1. 骨架屏 (Skeleton)    → 首帧即时渲染
2. Streaming SSR         → 内容分块到达
3. 渐进图片加载          → blur → 清晰
4. 无限滚动              → 列表页分页
5. 乐观更新              → 表单操作

加载状态优先级：
Content First → Layout → Images → Interactions
```

### 3.3 空状态与错误状态

| 场景 | 设计处理 |
|------|---------|
| 搜索无结果 | 插画 + 建议更换关键词提示 |
| 列表为空 | 引导创建/发现内容的提示 |
| 网络错误 | 友好提示 + 重试按钮 |
| 404 页面 | 插画 + 返回首页链接 |
| 权限不足 | 提示 + 登录引导 |
| 表单提交失败 | 保留已填内容 + 错误高亮 |

## 四、无障碍设计

- 所有可交互元素支持键盘导航（Tab 顺序 + Enter/Space 触发）
- 图片必须有 `alt` 文本
- 表单输入必须有 `label` 关联
- 颜色对比度达到 WCAG AA 标准（4.5:1 正文 / 3:1 大文本）
- 焦点状态可见（Focus Ring 样式）
- 语义化 HTML 标签（`<nav>`、`<main>`、`<article>`、`<section>`、`<aside>`）
- ARIA 角色标注（`role="navigation"`、`role="dialog"` 等）

## 五、前端性能预算

| 指标 | 目标 | 超限措施 |
|------|------|---------|
| 首屏 JS Bundle | < 150KB | 代码分割 + 动态导入 |
| 首屏 CSS | < 50KB | Tailwind JIT + Purge |
| 页面总大小 | < 500KB | 图片压缩 + 按需加载 |
| HTTP 请求数 | < 20 | 资源合并 + Preconnect |
| Lighthouse 分数 | > 90 | CI 门禁 |
