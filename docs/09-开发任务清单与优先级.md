# 芯师爷官网 — 详细开发任务清单（按优先级排序）

> 本文与 `06-项目开发计划.md`、`02-产品需求规格说明书.md`、`08-首页开发结构说明.md` 对齐；**序号越小优先级越高**。同一大级内按推荐实现顺序排列。

---

## 优先级定义

| 等级 | 含义 | 策略 |
|------|------|------|
| **P0** | 不做则无法演示「可用产品」或无法安全上线 | 必须最先完成 |
| **P1** | 核心业务闭环与主要获客路径 | P0 完成后立即做 |
| **P2** | 商业化与运营效率显著提升 | 有资源时并行或紧随 P1 |
| **P3** | 体验增强、国际化、高级能力 | 里程碑稳定后迭代 |

**依赖原则**：数据库与认证先于受保护页面；公共前台先于后台；读路径先于写路径；人工审核类功能需先有运营后台或种子数据。

---

## P0 — 基础设施与「能跑起来的站点」

| 顺序 | 任务 ID | 任务描述 | 交付物 / 验收 | 依赖 |
|------|---------|----------|---------------|------|
| P0-1 | `infra-repo` | 仓库初始化：单一 `package.json`、锁文件、脚本（`dev`/`build`/`lint`/`test`） | `npm ci` 成功 | 无 |
| P0-2 | `infra-docker` | `docker-compose`：PostgreSQL（+ 可选 Redis）单一定义、文档说明端口与卷 | 本地一键起库 | P0-1 |
| P0-3 | `infra-env` | `.env.example`、环境变量校验（如 `@t3-oss/env-nextjs` 或自研 Zod） | 缺变量时构建失败或清晰报错 | P0-1 |
| P0-4 | `infra-quality` | ESLint + Prettier + TypeScript `strict` + Husky + lint-staged + commitlint | PR 前自动检查 | P0-1 |
| P0-5 | `infra-ci` | GitHub Actions：lint + typecheck + test（可先空测） | main 保护分支通过 CI | P0-4 |
| P0-6 | `db-prisma-core` | Prisma 初始化；核心模型首版：`User`、`Session`（或等价）、`Company`、`Article`（或 `Post`）最小字段 | `migrate` + `seed` 可跑 | P0-2 |
| P0-7 | `db-seed` | 种子数据：若干企业、若干文章、一管理员账号（仅开发/Staging） | 首页/列表有真实形态数据 | P0-6 |
| P0-8 | `layout-shell` | 全站 `(public)/layout`：`SiteHeader`、`SiteFooter`、主内容区、404 基础页 | 任意路由有统一壳 | P0-1 |
| P0-9 | `page-home-skeleton` | 首页按 `08-首页开发结构说明` 搭 **静态/假数据** 区块骨架（含 `id` 与 `scroll-margin`） | `/` 可展示完整信息流结构 | P0-8 |
| P0-10 | `page-home-data` | 首页区块对接真实查询：Hero 文案配置、头条、Feed 前 N 条、推荐企业 | RSC 并行拉取、无瀑布阻塞首屏关键路径 | P0-7, P0-9 |
| P0-11 | `seo-metadata` | `generateMetadata`（title/description/OG）、全站 `robots.txt`、`sitemap` 雏形 | 首页可被正确分享预览 | P0-9 |
| P0-12 | `legal-about` | `/about`：平台介绍 + 联系方式（PRD P0） | 页脚与 CTA 有合法落地页 | P0-8 |

**P0 里程碑**：本地可开发、CI 绿、带种子数据的 **可浏览首页** + 关于页 + 基础 SEO。

## P1 — 核心门户：内容、企业、账号与基础搜索

| 顺序 | 任务 ID | 任务描述 | 交付物 / 验收 | 依赖 |
|------|---------|----------|---------------|------|
| P1-1 | `nav-subnav` | 首页 `HomeStickySubNav`：锚点跳转、`scroll-margin-top`、移动端横向滚动 | 与设计文档一致的可访问导航 | P0-9 |
| P1-2 | `home-sections` | 补齐首页其余区块：`HomeEvents`、`HomeAwards`、`HomeTopics`、`HomeCta`；`HomeProof` 可占位 | 与 `08` 文档区块顺序一致 | P0-10 |
| P1-3 | `home-ad-slot` | 首页可配置广告位（Banner 或 Feed 插卡），无配置时不占位错误 | 运营可关 | P0-6 需 `Ad`/`AdPosition` 最小模型或 JSON 配置 MVP |
| P1-4 | `articles-list` | `/articles`：列表、分类 Tab、分页、空状态 | PRD F-03 | P0-6 |
| P1-5 | `articles-detail` | `/articles/[slug]`：Markdown 渲染、元信息、相关文章推荐 | PRD F-03；`h1` 唯一 | P0-6 |
| P1-6 | `companies-list` | `/companies`：列表、筛选、分页、搜索（可先 URL query + DB `ilike`） | PRD F-02 | P0-6 |
| P1-7 | `companies-detail` | `/companies/[slug]`：Banner、介绍、Tab（产品/新闻/招聘占位） | PRD F-02 主路径 | P0-6 |
| P1-8 | `auth-register-login` | 用户注册/登录（邮箱+密码或项目选定方案）、HttpOnly Cookie 或安全存储 JWT | PRD 安全要求 | P0-6 |
| P1-9 | `auth-middleware` | `middleware`：保护 `/enterprise/*`、`/admin/*`；角色粗分 | 未登录不可进后台 | P1-8 |
| P1-10 | `search-mvp` | `/search`：先 **DB 查询** 或简单 `LIKE`/`pg_trgm`；结果分 Tab（企业/文章/产品可选） | PRD F-09 可用版 | P0-6 |
| P1-11 | `image-r2` | 上传走 R2（或 S3 兼容）+ 图片域名允许列表；文章/企业封面可用 | 无外链盗链风险 | P0-3 |
| P1-12 | `security-headers` | CSP（可分阶段收紧）、HSTS、X-Frame-Options 等 | 与 `07-部署与运维方案` 一致 | P0-8 |

**P1 里程碑**：**门户主路径完整**（首页 → 资讯 → 企业 → 搜索 → 关于），用户可注册登录，后台路由受保护。

| P2-11 | `admin-ads-awards` | 广告位配置、评选配置、投票数据导出或查看 | PRD A-01-06/08 | P2-9, P2-2, P2-5 |

**P2 里程碑**：**B 端可入驻、C 端可参与活动/投票/投递**，运营可审核与配置广告/评选。

## P3 — 体验、增长与工程化深化

| 顺序 | 任务 ID | 任务描述 | 交付物 / 验收 | 依赖 |
|------|---------|----------|---------------|------|
| P3-1 | `search-meili` | Meilisearch 接入、增量索引任务、中文分词调优 | 替代或增强 P1-10 | P2-10 稳定后更佳 |
| P3-2 | `i18n` | `next-intl`（或既定方案）：`/zh` `/en`、文案抽取、日期数字本地化 | PRD 3.5 | P1 前台页面 |
| P3-3 | `analytics` | Plausible/Umami 或同类；关键转化事件（注册、投递、投票） | 可看漏斗 | P0-8 |
| P3-4 | `observability` | Sentry、服务端结构化日志、核心 API P95 指标 | 与 `07` 对齐 | P0-5 |
| P3-5 | `test-e2e` | Playwright：首页、文章详情、登录、企业入驻主路径 | CI  nightly 或 PR 可选 | P1, P2 主路径 |
| P3-6 | `perf-budget` | Lighthouse CI 门禁、图片策略、`@next/bundle-analyzer` | 达 PRD 性能目标 | P1-2 |
| P3-7 | `a11y` | 焦点环、Skip link、对比度抽检、关键表单 `aria` | WCAG AA 抽检通过 | 全站 |
| P3-8 | `payments-report` | 报告付费下载（Stripe/国内支付二选一） | PRD 付费路径 | P2-4 |
| P3-9 | `enterprise-analytics` | 企业端数据看板（曝光、阅读、广告）与 `DataMetric` 写入任务 | PRD B-02-08 | P2-5, P0-6 |
| P3-10 | `media-soft-article` | 软文多渠道字段、运营回填阅读量（或爬虫二期） | PRD 软文矩阵 MVP | P2-8 |

**P3 里程碑**：可观测、可国际化、可测、性能与无障碍达标；付费与数据产品按需上线。
