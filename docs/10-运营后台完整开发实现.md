  id        String   @id @default(cuid())

  value     Json
  createdAt DateTime @default(now())

**AuditLog （操作审计）**
```prisma
model AuditLog {
  id         String   @id @default(cuid())
  userId     String?
  userEmail  String?
  action     String
  resource   String
  resourceId String?
  detail     Json?
  createdAt  DateTime @default(now())
  @@index([createdAt])
  @@index([userId])
  @@index([resource, resourceId])
}
```

## 四、JWT Session 扩展

原 session.ts 的 SessionPayload 新增 `adminRole` 字段：

```typescript
export type SessionPayload = {
  sub: string;
  role: UserRole;
  email: string;
  adminRole?: AdminRole | null;
};
```

登录 API (`/api/auth/login`) 将 `user.adminRole` 注入 JWT：

```typescript
const token = await signSessionToken({
  sub: user.id,
  role: user.role,
  email: user.email,
  adminRole: user.role === "ADMIN" ? user.adminRole : null,
});
```

## 五、UI 设计规范

### 色彩体系
- **背景**: `slate-950` (#020617) — 全后台统一
- **表面**: `white/5` ~ `white/10` — 卡片、输入框、表格行
- **主内容**: `white` — 标题、重要数据
- **辅助内容**: `slate-300` (#CBD5E1) — 表格正文
- **次要标签**: `slate-500` (#64748B) — 字段标签、描述
- **强调色**: `cyan-400/500/600` — 按钮、链接、active 指示器
- **成功**: `emerald-400/500/600`
- **警告**: `amber/yellow-400/500`
- **错误**: `red-400/500/600`

### 圆角规范
- 容器/卡片: `rounded-2xl`
- 内层面板/按钮: `rounded-xl`
- 表单控件: `rounded-lg`
- 标签/小元素: `rounded-full`

### 间距规范
- 页面 padding: `p-4 md:p-8`
- 卡片 padding: `p-5` ~ `p-6`
- 表单 field: `mt-4 space-y-4`
- 表格 cell: `px-4 py-3`

### 交互微动效
- hover: `hover:bg-white/5` 微高亮
- focus: `focus:border-cyan-500/50` 渐现边框
- Toast 入场: `animate-slide-up` (xe-slide-up 0.3s cubic-bezier)
- 导航 active: 左侧竖条 `before:absolute before:left-0 before:top-1/2 before:h-4 before:w-0.5 before:-translate-y-1/2 before:rounded-full before:bg-cyan-400`

## 六、关键设计决策

1. **CSR vs SSR**: 后台 Server Components (`force-dynamic`)，表单为 Client Components，布局为 RSC
2. **DataTable 列配置化**: `columns: [{ key, label, sortable, render }]`，分页由后端 `skip/take` 实现
3. **表单复用**: 新建和编辑共用同一 Form 组件，通过 `initialData` prop 区分
4. **Toast 体系**: `useReducer` 管理的全局 Context，包裹在 AdminLayout 中
5. **审核流程**: 企业详情页中嵌入审核面板，新建企业页面直接创建
6. **文章状态流转**: DRAFT → PENDING_REVIEW → APPROVED → PUBLISHED → ARCHIVED
7. **软删除**: 所有数据模型均使用 `deletedAt` 字段实现软删除
8. **操作日志**: 预留 AuditLog 模型，当前未自动接入（需后续添加中间件）

## 八、缺失与后续优化方向

### 已知缺失
1. **AuditLog 自动接入** — 需要在所有 mutation API 末尾手动 `prisma.auditLog.create()` 或使用 `prisma.$use` 中间件
2. **Markdown 编辑器** — 目前使用原生的 `<textarea>`，计划集成 `@uiw/react-md-editor`
3. **图表** — 计划使用 `recharts` 在 Dashboard 中添加趋势图表
4. **文章预览** — `/admin/articles/[id]/preview` 页面已预留路由但未实现
5. **图片上传** — `/api/admin/upload` 为占位，需集成 S3 或 local `public/uploads`

### 推荐后续优化
1. 添加 `prisma.$use` 中间件自动写入操作日志
2. 集成 `@uiw/react-md-editor` 富文本编辑器
3. 添加 Dashboard 趋势图表（recharts）
4. 实现文章预览页面
5. 图片上传功能（S3/本地）
6. 添加批量操作（批量发布、批量删除）
7. 添加更多的筛选条件（日期范围选择器等）
8. 性能优化：添加 SWR/React Query 数据缓存
9. 企业端后台功能扩展：企业管理自己的产品、文章、活动
